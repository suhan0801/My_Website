筆記與 AP325 Python 教材的關聯

以下筆記的順序與吳邦一教授的 AP325 Python 教材會非常相似，因為我大約 80% 的演算法知識都是從該教材中學習的，不過我的內容會跟原本的不一樣，因為我的目的是理解演算法而不是競賽。非常感謝吳邦一教授，提供了免費且內容完整的教材，幫助了像我這樣不知如何入門的學習者。

遞迴

概念

簡單來說，遞迴就是在一個函式中呼叫自己，並不斷進行相同的運算，直到滿足設定的終止條件，再一層一層返回最終結果。我們可以用一個簡單的例子來說明遞迴的運作方式：計算階乘。

實例


def factorial(n):
    # 基礎情況：當 n 為 0 或 1 時返回 1
    if n == 0 or n == 1:
        return 1
    
    # 遞迴步驟：n * (n-1) 的階乘
    return n * factorial(n - 1)

# 主程式，用來測試階層函數
number = int(input("請輸入一個非負整數: "))
result = factorial(number)
print(f"{number}! = {result}")

假設我們輸入數字 2，程式會先判斷該數字是否為 0 或 1。如果不是，則執行 n * factorial(n - 1)。此時 n 的值為 2，當程式執行到 factorial(n - 1) 時，遞迴再次調用該函數，此時 n 的值變為 1。由於 n 等於 1，滿足了終止條件，遞迴開始返回結果。這樣，前一層的 factorial(n - 1) 結果已知為 1，將其乘以當前層的 n（即 2），最終得到答案 2×1=2。這就是遞迴的基本運作原理。

優缺點

1.優點

簡潔性：遞迴可以使程式結構更加簡潔明了，特別適用於處理好幾層結構的問題。

2.缺點

空間問題：每次遞迴調用都會佔用一定的系統資源，當遞迴深度過深時，可能會導致「堆疊溢位」（Stack Overflow）問題。
效率問題：在一些情況下，遞迴可能會導致重複計算，降低效率。這在競賽或實際應用中可能導致「時間限制超過」（Time Limit Exceeded，縮寫為 TLE）。不過，這種情況是可以透過一些方法解決的，後續我會在文章中詳細討論。

結論

雖然遞迴看起來存在不少缺點，但這些問題都是可以透過優化來解決的，許多遞迴的變體或優化方法正是建立在遞迴的基礎上。因此，遞迴仍然是一個非常重要的概念。

心得

剛開始學習遞迴時，我覺得這個概念很抽象，而網路上的人都說遞迴非常簡單，這讓剛踏入這個領域的我感到壓力很大。為了克服這種困難，我反覆閱讀教材，並不斷練習題目。遇到解不出的問題時，我會去找解答，反覆操作了好幾次後，才逐漸掌握遞迴的核心概念。
